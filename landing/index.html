start building

Three steps. No local setup.

You bring an idea. The agent brings everything else.

step 01

Open it in Claude Code

Open the repo and pick the Sonnet 4.6 model. There's nothing to install first — the agent pulls Node, the SDK, and the deploy tools as it works.

step 02

Run /goal with your idea

Describe the app in a sentence. The agent brainstorms it, plans it, builds it, and shows you a live preview — one feature at a time.

step 03

Ship it with /deploy

The agent builds, uploads to decentralized storage, and registers your .dot name. You just approve the deploy on your phone.

🤖

You never touch a package manager or a deploy script. Agents install every prerequisite and run the deployment themselves — you describe the outcome you want, they handle the machinery and hand back a live URL.

what's wired up

Who said blockchain engineering?

The repo has everything you need and a tested agentic harness. Just ship.

// host api

Accounts & signing

Login and on-device approval through the Polkadot host — no browser wallets, no key handling in the app.

// product-sdk

Chain & storage

Reads, writes, and decentralized file storage through @parity/product-sdk — never a raw node endpoint.

// contract

On-chain leaderboard

A Rust → PolkaVM contract scaffold and a CDM manifest, ready to deploy under your own name.

// commands

/goal · /dev · /deploy

Build, preview, and publish slash-commands the agent drives end to end.

// subagents

Built-in reviewer

Fast Haiku subagents build scoped changes and check every diff against the Polkadot conventions.

// skills

SDK knowledge on tap

Per-package SDK guides are fetched into the project so the agent codes against the real API, not guesses.

prompt like a pro

Get more from every /goal.

  • Lead with /goal and one clear sentence, then let the agent ask the follow-ups.
  • Say what to keep and what to change — e.g. keep the leaderboard and storage, swap the game.
  • Build one feature at a time and preview with /dev between changes.
  • If the agent ever reaches for a browser wallet or a direct RPC URL, that's wrong for this template — tell it to use the Host API.
  • Rewrite the README to describe your app before /deploy — it becomes your app's public detail page.
